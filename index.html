UNIVERSAL GATES

NAND and NOR are functionally complete — every other gate, and therefore any Boolean function, can be built from NAND alone (or NOR alone). This is why standard‑cell libraries are built almost entirely from NAND/NOR stacks.

1.4 Boolean Algebra

The laws that let you simplify a logic expression before you ever cut silicon.

LawIdentity
IdentityA + 0 = A  ·  A·1 = A
NullA + 1 = 1  ·  A·0 = 0
IdempotentA + A = A  ·  A·A = A
ComplementA + A' = 1  ·  A·A' = 0
CommutativeA+B = B+A
Associative(A+B)+C = A+(B+C)
DistributiveA(B+C) = AB+AC
AbsorptionA + AB = A
DE MORGAN'S THEOREM
(A·B)'  =  A' + B'
(A+B)'  =  A'·B'

Break a bar over a junction and the operator underneath flips — AND becomes OR, OR becomes AND. This single rule is how every NAND/NOR‑only design is derived.

CONSENSUS & DUALITY

Consensus: AB + A'C + BC = AB + A'C — the BC term is redundant once both A‑paths exist. Duality: swap every + ↔ · and 0 ↔ 1 in a true identity and it stays true.

Module 02

Combinational Circuits

Outputs that depend only on the current inputs — no memory, no clock. Build them from gates, then never think about gates again.

2.1 Arithmetic Circuits — Adder / Subtractor

A full adder is just two half adders chained through the carry. Toggle A, B and Carry‑in.

Ripple‑carry chains a full adder per bit — simple, but the carry must propagate through every stage. Carry‑Look‑Ahead pre‑computes carries in parallel for speed; Carry‑Save and Wallace‑Tree adders defer carry resolution entirely to speed up multi‑operand addition (e.g. inside multipliers).

Module 04

Timing Concepts

A gate is never instant. Once delay enters the picture, so do setup time, hold time, and the clock's own imperfections.

4.1 Propagation Delay & Setup/Hold

Data (D) must arrive and settle before the clock edge (setup time) and stay stable after it (hold time) — violate either window and the flip‑flop's output goes metastable.

CLOCK SKEW

Difference in clock arrival time at two flip‑flops fed from the same source — caused by unequal wire/buffer delay.

CLOCK JITTER

Cycle‑to‑cycle variation in the clock edge's exact timing — a noise term, not a fixed offset.

DUTY CYCLE

Fraction of one clock period spent high — 50% is ideal for most synchronous logic.

4.2 Metastability & Clock‑Domain Crossing

When setup/hold is violated, a flip‑flop's output can hover between 0 and 1 before resolving — unpredictably. Watch it happen below.

The fix: a two‑flop synchroniser — pass the async signal through two back‑to‑back flip‑flops on the receiving clock, giving any metastable glitch a full cycle to settle before it's used.

Module 07

Verilog — Language Basics

Verilog describes hardware, it doesn't execute line‑by‑line like software. A module is a black box; everything inside fires in parallel.

7.1 Module Structure

module full_adder (
  input  a, b, cin,
  output sum, cout
);
  parameter WIDTH = 1; // compile-time constant

  assign sum  = a ^ b ^ cin;
  assign cout = (a&b) | (b&cin) | (a&cin);
endmodule

Every module has a name, a port list (input / output / inout), and a body. parameter values can be overridden at instantiation, letting one module describe a whole family of widths.

7.4 Procedural Blocks

INITIAL — runs once 
initial begin
  clk = 0;
  rst = 1;
end
ALWAYS — re‑triggers forever 
always @(*)            // combinational
  y = a & b;
always @(posedge clk)  // sequential
  q <= d;

7.5 Control Flow — Conditionals & Loops

CASE / CASEX / CASEZ 
case (sel)
  2'b00: y = in0;
  2'b01: y = in1;
  default: y = 0;
endcase
FOR LOOP 
integer i;
for (i=0; i<8; i=i+1)
  mem[i] = 0;

while / repeat / forever round out the loop set; generate for is the elaboration‑time loop covered in 7.7 below — it instantiates hardware, it doesn't run in time.

7.6 Tasks & Functions

FUNCTION — returns a value, zero delay 
function [7:0] add8;
  input [7:0] x, y;
  add8 = x + y;
endfunction
TASK — can consume time, no return value 
task pulse_clk;
  begin
    clk = 0; #5;
    clk = 1; #5;
  end
endtask
